Site Updates

I'm giving learning in public a try

Accessibility matters

I've been making small tweaks throughout the site lately.

A lot of it has to do with accessibility--better labeling in HTML, more semantic HTML in general, and using ARIA controls to better define elements for screen readers.

Most recently, that includes a brand new "jump to main content" button, plus read times, section links, and fewer big animations in my blog posts.

Another blog post appears

After a month recapping the process of publishing a Python library, I compiled my journey and released it to my blog. You can read my latest blog post here.

I took a much different approach to this post than the last, since I'm still experimenting. It was a lot of work, and reviewing the work I'd done ended up helping me learn from my experience and grow from it.

Back in a new form

I'm no longer using a 3rd-party plugin to automate my forms. They're all me now! Well, and a 3rd party endpoint provided by FormSpree

It ended up being a bigger undertaking than I'd expected, but I'm happy to be rid of form-builders (hopefully) forever

Redefining content

I cut about 25% of the words on the site. No, really. This decision came from advice I'd gotten about website copy, but also from my intuition after interacting with the site personally

Nobody likes blocks of text. So I'm trying to refine the copy so you know what's important sooner, and don't have to sweat the smalls stuff

Brand new colors ... sorta

First off, I removed the header socials. They were distracting

Next, I converted the whole site to using an hsl palette with css variables, which makes tweaking my colors way easier since I seem prone to obsessing over them anyway

Mostly, the color theme is the same. Dark theme is still a work in progress. I changed where the emphasis lays on each page, and tightened up my color usage so it's more consistent throughout

Rewriting everything

...And they said copy was easy

Actually, I don't think anyone's said that. Because wow, I've been tinkering a lot and still feel like the copy is not quite me enough

Anyway, all the words are different. If you want to see what changed, feel free to visit the source on GitHub

My first blog post is live!

After fixing an enormous merge issue from not properly maintaining my branches in git (oops), my first blog post is here!

I'm so excited to be officially producing content, and phew is it tougher than I'd anticipated. I'm used to a healthy share of creative writing but blogging feels way more like my APA-fueled Psychology research papers than I want to admit

Simplified social menus

Previous versions of the site had an always-on social media bar across the top of each page, just below the header

It looked cluttered so I hid them. Now you can use the arrow button to toggle their visibility. Just hit the

Semantic coding

Articles are sections, and sections are articles! You won't notice a visual change, but if you inspect the site you'll see an improvement to the semantics of the HTML structure overall

Mainly, I'm making a more concerted effort to use the right tools for the job. Basically, major components are now wrapped as articles instead of as sections, and each article has its own sections

Snazzy scrolling

I smoothed out page transitions so you don't get whiplash whilst zooming around the home page or returning to the header

Now that the design is (mostly) figured out, I can start adding fun features and making the site dynamic. Look forward to more interactivity and design as I continue to polish each page
